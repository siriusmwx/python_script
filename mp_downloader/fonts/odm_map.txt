用法：运行 [选项] <数据集名称>

ODM 是一个命令行工具包，用于从无人机、气球或风筝图像生成地图、点云、3D 模型和
DEM。

位置参数：
<数据集名称> 数据集的名称（即项目文件夹中的子文件夹名称）。默认值：code

可选参数：
-h, --help 显示此帮助消息并退出
--project-path <路径>
                    项目文件夹的路径。您的项目文件夹应
                    包含每个数据集的子文件夹。每个数据集
                    都应有一个"images"文件夹。
--end-with <字符串>, -e <字符串>
                    在此阶段结束处理。可以是以下之一：数据集、
                    split、merge、opensfm、openmvs、odm_filterpoints、
                    odm_meshing、mvs_texturing、odm_georeferencing、
                    odm_dem、odm_orthophoto、odm_report、odm_postprocess。
                    默认值：odm_postprocess
--rerun <string>, -r <string>
                    仅重新运行此阶段并停止。可以是以下之一：
                    dataset、split、merge、opensfm、openmvs、
                    odm_filterpoints、odm_meshing、mvs_texturing、
                    odm_georeferencing、odm_dem、odm_orthophoto、
                    odm_report、odm_postprocess。默认值：None
--rerun-all         永久删除所有先前的结果并重新运行处理管道。
--rerun-from <string>
                    从此阶段重新运行处理。可以是以下之一：
                    dataset、split、merge、opensfm、openmvs、
                    odm_filterpoints、odm_meshing、mvs_texturing、
                    odm_georeferencing、odm_dem、odm_orthophoto、
                    odm_report、odm_postprocess。默认值：None
--min-num-features <integer>
                    每幅图像提取的最小特征数。更多特征可用于在图像之间
                    找到更多匹配，从而可能允许重建重叠很少或特征不足的区域。
                    更多特征也会减慢处理速度。默认值：10000
--feature-type <string>
                    选择提取关键点和计算描述符的算法。可以是以下之一：
                    akaze、dspsift、hahog、orb、sift。默认值：dspsift
--feature-quality <string>
                    设置特征提取质量。更高的质​​量会生成更好的特征，
                    但需要更多内存并且需要更长时间。可以是以下之一：
                    ultra,high,medium,low,lowest。默认值：high
--matcher-type <string>
                    匹配器算法，用于近似最近邻或词袋的快速库。
                    FLANN 较慢，但更稳定。BOW 更快，但有时会错过
                    有效匹配。 BRUTEFORCE 非常慢但很强大。可以是
                    bow、bruteforce、flann之一。默认值：flann
--matcher-neighbors <正整数>
                    根据 GPS exif 数据与最近的图像执行图像匹配。设置为 0 表示通过三角测量匹配。
                    默认值：0
--matcher-order <正整数>
                    根据图像文件名顺序与最近的 N 个图像执行图像匹配。
                    可以加快处理连续图像的速度，例如从视频中提取的图像。
                    它仅适用于非地理参考数据集。设置为 0 表示禁用。默认值：0
--use-fixed-camera-params
                    在捆绑调整期间关闭相机参数优化。这有时可用于改善
                    表现出圆顶/保龄球形或使用滚动快门相机拍摄的图像
                    的结果。默认值：False
--cameras <json> 
                    使用从另一个数据集计算的相机参数，而不是计算它们。可以指定为
                    cameras.json 文件的路径，也可以指定为表示 cameras.json
                    文件内容的 JSON字符串。默认值：
--camera-lens <string>
                    设置相机投影类型。手动设置值可以帮助改善几何不失真。
                    默认情况下，应用程序会尝试从图像元数据中确定镜头类型。
                    可以是以下之一：自动auto、透视perspective、棕色brown、
                    鱼眼fisheye、fisheye_opencv、球面spherical、
                    等距equirectangular，dual。默认值：auto
--radiometric-calibration <string>
                    设置要对图像执行的辐射校准。处理多光谱和热图像时，您应该设置此选项以获取
                    反射率/温度值（否则您将获得数字值）。[camera] 应用黑电平、晕影、行梯度增
                    益/曝光补偿（如果找到适当的 EXIF 标签）并计算绝对温度值。[camera+sun] 是
                    实验性的，应用 [camera] 的所有校正，并补偿通过下行光传感器 (DLS) 记录的
                    光谱辐射，同时考虑太阳的角度。可以是以下之一：none、camera、camera+sun。默认值：none
--max-concurrency <positive integer>
                    各个进程中使用的最大进程数。峰值内存要求为每个线程约1GB
                    和2百万像素图像分辨率。默认值：12
--use-hybrid-bundle-adjustment
                    对每个添加到重建的图像运行局部束调整，每 100 个图像运行全局调整。
                    加快非常大的数据集的重建速度。默认值：False
--sfm-algorithm <string>
                    选择运动结构算法。对于空中数据集，如果相机 GPS 位置和角度可用，
                    三角测量可以产生更好的结果。对于在固定高度使用仅有地面图像的平
                    面场景，平面可以更快。可以是以下之一：incremental、
                    triangulation、planar。默认值：incremental
--sfm-no-partial 
                    不要尝试合并部分重建。当图像没有足够的重叠或被孤立时，
                    可能会发生这种情况。默认值：False
--sky-removal       使用 AI 自动计算图像蒙版以去除天空。实验。默认值：False
--bg-removal        使用 AI 自动计算图像蒙版以去除背景。实验。默认值：False
--use-3dmesh        使用完整的 3D 网格来计算正射影像，而不是 2.5D 网格。
                    此选项速度更快，并且在平面区域提供类似的结果。默认值：False
--skip-3dmodel      跳过完整 3D 模型的生成。如果您只需要 2D 结果（例如正射影像和 DEM），
                    这可以节省时间。默认值：False
--skip-report       跳过 PDF 报告的生成。如果您不需要报告，这可以节省时间。默认值：False
--skip-orthophoto   跳过正射影像的生成。如果您只需要 3D 结果或 DEM，这可以节省时间。默认值：False
--ignore-gsd        忽略地面采样距离 (GSD)。如果设置为 true，则与默认行为相比，内存和处理器消耗会发生变化。通常，GSD 估计值用于限制图像输出的最大分辨率，并在必要时调整图像大小，从而加快处理速度并降低内存使用率。由于 GSD 是估计值，因此有时忽略它可以稍微提高图像输出质量。除非您确定需要它，否则切勿将 --ignore-gsd 设置为 true，即使这样也不要使用它。默认值：False
--no-gpu            即使可用，也不要使用 GPU 加速。默认值：False
--mesh-size <整数>  输出网格的最大顶点数。默认值：200000
--mesh-octree-depth <整数：1 <= x <= 14>
                    网格重建中使用的八叉树深度，增加可获得更多顶点，建议值为 8-12。默认值：11
--fast-orthophoto   跳过密集重建和 3D 模型生成。它直接从稀疏重建中生成正射影像。
                    如果您只需要正射影像而不需要完整的 3D 模型，请打开此选项。默认值：False
--crop <positive float>
                    自动裁剪通过在数据集边界周围创建一个平滑的缓冲区来限制图像
                    输出，缩小 N 米。使用 0 禁用裁剪。默认值：3
--boundary <json> 
                    GeoJSON 多边形限制重建的区域。可以指定为 GeoJSON 文件的路径，
                    也可以指定为表示 GeoJSON 文件内容的 JSON 字符串。默认值：
--auto-boundary 
                    使用相机拍摄位置自动设置边界以限制重建的区域。这有助于消除远处的
                    背景伪影（天空、背景景观等）。另请参阅 --boundary。默认值：False
--auto-boundary-distance <positive float>
                    使用 --auto-boundary 计算边界时，指定相机拍摄位置与边界外边缘
                    之间的距离。设置为 0 以自动选择一个值。默认值：0
--pc-quality <string>
                    设置点云质量。更高的质​​量会生成更好、更密集的点云，但需要更多
                    内存并且花费更长时间。质量每提高一步，处理时间大约增加 4 倍。
                    可以是以下之一：ultra, high, medium, low, lowest。默认值：medium
--pc-classify 
                    对点云输出进行分类。您可以通过调整 --dem-* 参数来控制
                    此选项的行为。默认值：False
--pc-csv            以CSV格式导出地理参考点云。默认值：False
--pc-las            以LAS格式导出地理参考点云。默认值：False
--pc-ept            以Entwine Point Tile(EPT)格式导出地理参考点云。默认值：False
--pc-copc           以云优化点云(COPC)格式保存地理参考点云。默认值：False
--pc-filter <positive float>
                    通过删除偏离局部平均值超过N个标准差的点来过滤点云。
                    设置为 0 以禁用过滤。默认值：5
--pc-sample <positive float>
                    通过仅保留半径 N（以米为单位）周围的单个点来过滤点云。
                    这可用于限制点云的输出分辨率和删除重复点。设置为 0 以
                    禁用采样。默认值：0
--pc-skip-geometric 
                    几何估计通过计算几何一致的深度图来提高点云的准确性但可能
                    无法用于较大的数据集。此标志禁用几何估计。默认值：False
--smrf-scalar <positive float>
                    简单形态滤波器高程标量参数。默认值：1.25
--smrf-slope <positive float>
                    简单形态滤波器斜率参数（上升/下降）。默认值：0.15
--smrf-threshold <positive float>
                    简单形态滤波器高程阈值参数（米）。默认值：0.5
--smrf-window <positive float>
                    简单形态滤波器窗口半径参数（米）。默认值：18.0
--texturing-skip-global-seam-leveling
                    跳过所有图像的颜色标准化。在处理辐射数据时很有用。默认值：False
--texturing-keep-unseen-faces
                    保留网格中任何相机中都看不到的面。默认值：False
--texturing-single-material
                    生成具有单一材质和单一纹理文件的 OBJ，而不是多个。默认值：False
--gltf              生成单个文件二进制 glTF (GLB) 纹理模型。默认值：False
--gcp <path string> 
                    包含用于地理配准的地面控制点的文件路径。该文件需要使用
                    以下格式：EPSG:<code> 或 <+proj definition>
                    geo_x geo_y geo_z im_x im_y image_name [gcp_name]
                    [extra1] [extra2] 默认值：无
--geo <path string> 
                    包含用于地理参考的相机中心坐标的图像地理定位文件路径。如果
                    您没有偏航/俯仰/滚动的值，您可以将其设置为0。该文件需要使用以下格式：
                    EPSG：<code> 或 <+proj definition> image_name geo_x geo_y geo_z 
                    [偏航（度）] [俯仰（度）] [滚动（度）] [水平精度（米）] [垂直精度（米）] 默认值：None
--align <path string>
                    指向 GeoTIFF DEM 或 LAS/LAZ 点云的路径，重建输出应自动与之对齐。实验性的。默认值：无
--use-exif          如果您有 GCP 文件但想使用 EXIF 信息进行地理配准，请使用此标签。默认值：False
--dtm               使用此标签使用简单的形态过滤器构建 DTM（数字地形模型，仅限地面）。
                    检查 --dem* 和 --smrf* 参数以进行更精细的调整。默认值：False
--dsm               使用此标签使用渐进形态滤波器构建 DSM（数字表面模型，地面 + 物体）。
检查 --dem* 参数以进行更精细的调整。默认值：False
--dem-gapfill-steps <正整数>
                    用于填充有间隙区域的步数。设置为 0以禁用间隙填充。从等于输出分辨率的半径开始，
                    使用反距离加权 (IDW) 算法生成半径逐渐增大的 N 个不同的 DEM 并合并在一起。
                    然后使用最近邻插值合并剩余的间隙。默认值：3
--dem-resolution <浮点>
                    DSM/DTM 分辨率（厘米/像素）。请注意，此值受地面采样距离 (GSD)
                    估计值的限制。默认值：5
--dem-decimation <正整数>
                    在生成 DEM 之前抽取点。1 表示不抽取（全质量）。100 表示抽取 ~99% 的点。
                    有助于加快生成非常大的数据集的 DEM 结果的速度。默认值：1
--dem-euclidean-map 
                    为每个 DEM 计算欧几里得栅格地图。地图报告从每个单元到最近的
                    NODATA 值的距离（在任何孔填充之前）。这对于隔离已填充的区域
                    很有用。默认值：False
--orthophoto-resolution <float > 0.0>
                    正射影像分辨率（厘米/像素）。请注意，此值受地面采样距离 (GSD) 估计值限制。默认值：5
--orthophoto-no-tiled
                    如果您想要条纹 GeoTIFF，请设置此参数。默认值：False
--orthophoto-png 
                    如果您想要生成正射影像的PNG渲染，请设置此参数。默认值：False
--orthophoto-kmz    如果要生成正射影像的 Google Earth (KMZ) 渲染，请设置此参数。默认值：False
--orthophoto-compression <string>
                    设置正射影像使用的压缩。可以是以下之一：JPEG、LZW、PACKBITS、
                    DEFLATE、LZMA、NONE。默认值：DEFLATE
--orthophoto-cutline 
                    在裁剪区域周围生成一个多边形，该多边形会剪切特征边缘周围的正射影像。
                    此多边形可用于拼接具有多个重叠正射影像的无缝马赛克。默认值：False
--tiles             为正射影像和 DEM 生成静态图块，这些图块适用于 Leaflet 或 OpenLayers
                    等查看器。默认值：False
--3d-tiles          生成 OGC 3D 图块输出。默认值：False
--rolling-shutter 
                    启用滚动快门校正。如果相机具有滚动快门，并且图像是在运动中拍摄的，
                    您可以打开此选项以提高结果的准确性。另请参阅 --rolling-shutter-readout。
                    默认值：False
--rolling-shutter-readout <正整数>
                    覆盖相机传感器的滚动快门读出时间（以毫秒为单位），而不是使用
                    滚动快门读出数据库。请注意，并非所有
                    相机都存在于数据中基础。设置为 0 以使用数据库值。默认值：0
--build-overviews   构建正射影像概览，以便在 QGIS 等程序中更快地显示。默认值：False
--cog               创建云优化 GeoTIFF，而不是普通 GeoTIFF。默认值：False
--copy-to <path>    处理后将输出结果复制到此文件夹。
--version           显示版本号并退出。
--video-limit <正整数>
                    处理时从视频文件中提取的最大帧数。设置为 0 表示无限制。默认值：500
--video-resolution <正整数>
                    提取的视频帧的最大输出分辨率（以像素为单位）。默认值：4000
--split <正整数>
                    每个子模型的平均图像数。将大型数据集拆分为较小的子模型时，图像将分组为群集。
                    此值规定每个群集平均应具有的图像数量。默认值：999999
--split-overlap <正整数>
                    子模型之间重叠的半径。将图像分组到群集后，距离群集小于此半径的图像
                    将添加到群集。这样做是为了确保相邻的子模型重叠。默认值：150
--split-image-groups <路径字符串>
                    控制如何将图像分成组的图像组文件的路径。该文件需要
                    使用以下格式：image_name group_name 默认值：None
--sm-no-align 
                    跳过拆分合并中的子模型对齐。如果 GPS 在非常大的
                    数据集上足够好，则很有用。默认值：False
--sm-cluster <字符串>
                    用于在多个节点上并行分发拆分合并工作流的 ClusterODM 实例的 URL。默认值：无
--merge <字符串> 
                    选择在拆分数据集的合并步骤中合并的内容。默认情况下，所有可用的输出都会合并。
                    选项：all、pointcloud、orthophoto、dem。默认值：all
--force-gps 
                    使用图像的 GPS exif 数据进行重建，即使存在 GCP。如果您有高精度
                    GPS测量，此标志很有用。如果没有 GCP，此标志不起作用。默认值：False
--gps-accuracy <positive float>
                    为所有图像的 GPS 精度稀释 (DOP) 信息设置一个以米为单位的值。
                    如果您的图像标有高精度 GPS 信息 (RTK)，则将自动相应地设置此值。
                    您可以使用此选项在重建失败的情况下手动设置它。降低此选项有时
                    可以帮助控制大面积的保龄球效应。默认值：3
--optimize-disk-space
                    删除繁重的中间文件以优化磁盘空间使用率。这会影响从中间阶段重新启动
                    管道的能力，但允许在没有足够磁盘空间的机器上处理数据集。默认值：False
--pc-rectify 
                    对点云执行地面校正。这意味着错误分类的地面点将被重新分类，
                    空白将被填补。对于生成 DTM 很有用。默认值：False
--primary-band <string>
                    处理多光谱数据集时，您可以指定将用于重建的主波段的名称。
                    建议选择具有清晰细节且焦点清晰的波段。默认值：auto
--skip-band-alignment
                    处理多光谱数据集时，ODM 将自动对齐每个波段的图像。
                    如果图像已经过后处理并且已经对齐，请使用此选项。
                    默认值：False
